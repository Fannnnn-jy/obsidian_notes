# NeRF (Neural Radiance Fields, 2020)

- 核心理念：世界是一个连续的函数。
- 将整个 3D 场景压缩进一个神经网络（通常是多层感知机 MLP）的权重中。
- 工作原理（隐式表示）：
    1. 输入： 空间坐标 $(x, y, z)$ 和 观看方向 $(\theta, \phi)$。
    2. 黑盒处理： 将这些数据喂给一个 MLP 网络。
    3. 输出： 该点的 颜色 RGB 和 体密度（Opacity/Density, $\sigma$）**。        
    4. 渲染（体渲染 Volume Rendering）： 为了生成一张图片，NeRF 会从相机发出无数条光线（Ray Marching）。对于每一条光线，它在 3D 空间中采样数百个点，询问神经网络“这里有什么颜色？有多浓？”，最后通过积分将这些点的颜色合成，得到像素点的颜色。

- 优点：
    - 极高的精度： 能够极其逼真地还原光影、反射（如镜面）和半透明物体。
    - 存储极小： 一个巨大的场景可能只需要几 MB 的模型权重文件。
    - 连续性： 因为是数学函数，理论上可以无限放大分辨率。

- 缺点：
    - 训练慢： 也就是通常说的“炼丹”，需要数小时甚至数天。
    - 渲染极慢（推理慢）： 生成一张图需要查询网络数百万次（每条光线采样几百个点），导致无法实时运行（原始 NeRF 甚至只有 0.1 FPS）。
    - 黑盒： 很难直接编辑场景（比如“把这把椅子移走”），因为场景都在神经网络的权重里。

# Gaussian Splatting (3DGS, 2023)

- 核心理念：世界是由无数个带颜色的“模糊椭球”组成的。
- 放弃了繁重的神经网络推理，回归到了更接近传统图形学的“显式”表达，但做了一个巧妙的改动。

- 工作原理（显式/半显式表示）：
    1. 基本单元（3D 高斯球）： 它不再用神经网络，而是用数百万个**3D 高斯椭球体**来表示场景。你可以把它们想象成空间中漂浮的、半透明的、带颜色的“光团”。
    2. 属性： 每个高斯球只有这几个参数：位置、协方差（决定它是长条还是扁平、怎么旋转）、不透明度、以及球谐函数系数（Spherical Harmonics，用于表示不同角度看的颜色变化）。
    3. 渲染（Splatting 溅射）： 这就是它快的原因。它不需要沿着光线苦苦采样，而是直接把这些 3D 椭球投影（Rasterize）到 2D 屏幕上。这就好比把一个个颜料球“啪”地扔在画布上，重叠混合。因为利用了 GPU 高效的排序和光栅化管线，速度极快。
        
- 优点：
    - 实时渲染（极快）： 可以轻松达到 **100 FPS+** 甚至更高，完全满足 VR/AR 和游戏需求。
    - 训练快： 几分钟到几十分钟就能收敛。
    - 质量极高： 在细节保持上通常优于 NeRF。
    - 可编辑性： 因为是离散的点云，理论上可以直接删除或移动一部分高斯球。
        
- 缺点：
    - 显存占用大： 需要存储数百万个高斯球的参数，显存占用远高于 NeRF（GB 级别）。
    - 伪影： 在未被相机拍摄到的区域，可能会出现奇怪的针刺状或漂浮物。
